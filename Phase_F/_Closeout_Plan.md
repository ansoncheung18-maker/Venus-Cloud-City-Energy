## 1. 退役條件

喺以下任何一種情況下，任務將進入退役程序：

| # | 條件 | 說明 |
|:---|:---|:---|
| 1 | **設計壽命屆滿** | 營運滿 10-15 年 |
| 2 | **災難性故障** | 核心子系統無法修復 |
| 3 | **成本效益過低** | 年營運成本 > 年輸出價值 |
| 4 | **技術過時** | 新技術取代 |
| 5 | **嚴重腐蝕** | 結構完整性受損 |
| 6 | **預算中斷** | 資金無法持續 |

## 2. 關閉程序

### 2.1 關閉順序

| 步驟 | 動作 | 說明 |
|:---|:---|:---|
| 1 | 停止微波傳輸 | 通知接收衛星 |
| 2 | 關閉太陽能系統 | 斷開電路 |
| 3 | 切換至備用電源 | 維持控制系統 48 小時 |
| 4 | 下載最後數據 | 傳回地球存檔 |
| 5 | 排空氦氣 | 氣囊放氣 |
| 6 | 關閉冷卻系統 | 關閉所有系統 |
| 7 | 關閉控制系統 | 發送最後通訊 |
| 8 | 切斷所有電源 | 平台靜止 |

### 2.2 預計關閉時間

| 階段 | 時間 | 備註 |
|:---|:---|:---|
| 準備工作 | 1 週 | 通知、備份 |
| 關閉操作 | 24 小時 | 自動化執行 |
| 最終確認 | 48 小時 | 遠程檢查 |

## 5. 設備處置

### 5.1 處置選項

| 選項 | 說明 | 可行性 |
|:---|:---|:---|
| 1. 留原地 | 平台靜止，自然墜落 | ✅ 最簡單 |
| 2. 可控墜落 | 導引至金星表面 | ⚠️ 需額外燃料 |
| 3. 部分回收 | 回收有價值組件 | ❌ 成本極高 |
| 4. 大氣層燒毀 | 下降至低空燒毀 | ✅ 可行（需控制） |

### 5.2 推薦方案

**平台自然墜落，最終喺金星低空大氣層燒毀。**

| 優點 | 缺點 |
|:---|:---|
| 無需額外成本 | 無法回收 |
| 大氣層自然銷毀 | - |
| 無太空垃圾 | - |
| 無需複雜操作 | - |
